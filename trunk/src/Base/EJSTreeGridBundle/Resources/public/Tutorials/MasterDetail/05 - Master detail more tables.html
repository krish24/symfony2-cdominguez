  • One master grid can show its rows in more different detail grids, even every row in different detail grid.
    One master row can be show in more detail grids at once.
    In one detail grid can be shown data from more master grids, but not at once.
    Even detail grid can work also as master grid and show its rows in another detail grid.
  • This example shows two master grids 1 and 2. Every master grid shows every its row in next two detail grids 1 and 2. And the detail grids 1 and 2 show their rows in the last detail grid.
  • The data are always in the main master grids. The detail grids that works also as master grids do not contain any data, all their data are still copied from their masters.
  • The master row attribute

    Detail

    can contain more detail grids ids to be copies to all these grids.
  • To choose which master rows children will be copied to which detail grid define Cfg attribute

    MasterDef

    in detail grid, with list of row default names. Only row with Def attribute listed in MasterDef will be copied to given grid.
    It is even possible to copy one child row to more detail grids, the changes in such row will be mirrored among all the grids.
    If the detail grid works also as master grid for another detail, the defaults for the second detail must be listed in the MasterDef too (in this example the 'DETAIL' def).
